API Endpoints

Our endpoints provide the appropriate GET, POST, PUT & DELETE requests and responses for the many route parameters of our restaurant's API.

Parameters

Each route parameter are shown in the parameter drop down.

Request Body

The Request Body drop down of each endpoint showcases if a body is required and also what to include in the body of the respected request.

Status Codes

The Status Codes drop down of each endpoint showcases the status codes associated with each endpoint. You will also find a snippet of code as an example of a successful operation.

Endpoints

/food

Returns a list of all todos. However if an id is provided, it will only return the single requested id

id [number]: the id of the todo

Body Not Required

200 = Successful/OK
400 = Requested body not permitted
400 = Query Parameters do not meet requirements
400 = ID is not a number
400 = Todo with that ID not found

/food

Adds a todo to the list. If a list parameter is provided it adds multiple to dos in one call

id[number](required): the id of the task to be updated to othe todolist

todo[string](required): the task to be added to the todolist list[array]:contains a list of todos

200 = Successful/OK
400 = Requested body not permitted
400 = Query Parameters do not meet requirements
400 = ID is not a number
400 = Todo with that ID not found

/food/:id

Updates an existing to do and modifies its data

id[number](required): the id of the task to be updated to othe todolist

todo[string]:the task to be updated completed[boolean]: the status of the task

200 = Successful/OK
400 = Requested body not permitted
400 = Query Parameters do not meet requirements
400 = ID is not a number
400 = Todo with that ID not found

/food/:id

Deletes an existing to do

id[number](required): the id of the task to be updated to othe todolist

Body Not Permitted

200 = Successful/OK
400 = Requested body not permitted
400 = Query Parameters do not meet requirements
400 = ID is not a number
400 = Todo with that ID not found
